    # cookie

    cookie指的就是浏览器里能存储的一种数据, 仅仅是浏览器实现的一种数据存储功能. 它由服务器生成, 附着在http请求上, 浏览器把cookie以键值对形式保存在某个目录下的文本文件内. 可以携带用户信息, 当服务器检查cookie的时候, 便可以获取到客户端的状态.

    cookie的应用场景

    • 会话状态管理, 比如下次自动登录, 购物车
    • 个性化设置, 如用户自定义设置, 用户主题
    • 浏览器行为跟踪, 比如跟踪分析用户行为

    cookie的生成方式:

    • http response header中的set-cookie
    • js通过document.cookie读写cookie
    document.cookie = "userid=1";
    1

    设置cookie的过期时间, 通过前端手段设置cookie过期时间, 一定要设置成服务端时间, 不能使用客户端本地时间

    document.cookie = "userid=1; max-age=86400";
    1

    获取名称是_csrfToken的cookie值

    function getCookie(name) {
	let arr = document.cookie.split(/;\S/);
	for(let i = 0; i < arr.length; i++) {
		let params = arr[i].split("=");
		if(name === params[0]) {
			return params[1];
		}
		return '';
	}
}

    cookie的缺陷

    • cookie不够大, cookie的大小限制在4KB左右
    • 过多的cookie会带来巨大的性能浪费
    • 在http请求中cookie是明文传输的, 不安全

    # session

    session代表着服务器和客户端一次会话的过程.

    • session保存在服务器端, session的有效期比较短

    # LocalStorage

    • 保存的数据长期存在, 下一次访问该网站的时候, 网页可以直接读取以前保存的数据
    • 大小为5M左右, 保存的数据以键值对形式存在.
    • 仅在客户端使用, 不与服务端进行通信

    应用场景: 图片内容丰富的电商网站会用它来存储base64图片

    localStorage的过期判断逻辑

    const maxAge = 24 * 36e5 * 7; // 过期时间为7天

localStorage.setItem('userid', JSON.stringify({
	data: 1,
	stamp: Date.now + maxAge
}));

let store_userid = JSON.parse(localStorage.getItem('userid'));
if(store_userid <= Date.now()) {
	localStorage.removeItem('userid');
} else {
	let userid = store_userid;
}

    # SessionStorage

    保存的数据用于浏览器的一次会话, 当会话结束(通常是关闭窗口), 数据被清空. 即使是相同域名下的两个页面, 只要它们不在同一个浏览器窗口中打开, 那么它们的sessionStorage内容便无法共享. localStorage在所有同源窗口中都是共享的.
    大小为5M左右, 仅在客户端使用

    sessionStorage.setItem(name, value) // 保存数据

let myName = sessionStorage.key(index) // 根据索引获取数据名称

sessionStorage.getItem(myName) // 根据名称获取数据内容

sessionStorage.removeItem(myName)

sessionStorage.clear() // 清除全部数据
