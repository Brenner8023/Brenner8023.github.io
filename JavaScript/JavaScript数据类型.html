                            # undefined和null

                            对未初始化和未声明的变量执行typeof都会返回undefined

                            对于 null 来说，很多人会认为他是个对象类型，其实这是错误的。虽然 typeof null 会输出 object，但是这只是 JS 存在的一个悠久 Bug。在 JS 的最初版本中使用的是 32 位系统，为了性能考虑使用低位存储变量的类型信息，000 开头代表是对象，然而 null 表示为全零，所以将它错误的判断为 object 。虽然现在的内部类型判断代码已经改变了，但是对于这个 Bug 却是一直流传下来

                            # 类型判断

                            # typeof

                            • 对于非引用的基本类型, 除null以外, 均可返回意料之中的结果
                            • 对于引用类型, 除function以外, 一律返回object
                            • 对于null, 返回object
                            • 对于function, 返回function
                            • typeof(typeof a)返回的是string

                            # instanceof

                            A instanceof B用来检测A是否是B的实例, 是返回true, 否返回false. 不可用于检测非引用数据类型. 如果B的显式原型对象在A的原型链上, 那么A是B的实例, 返回true

                            # toString()

                            每一个引用类型都有toString()方法, 如果此方法在对象中未被覆盖重写, 则toString()返回"[object type]". 我们可以直接调用Object原型上未被覆盖的toString()方法Object.prototype.toString.call(true)

                            # ==

                            • NaN和任何类型比较都返回false
                            • boolean和其他类型比较, boolean首先被转换为number类型
                            • string和number比较, string转换为number类型
