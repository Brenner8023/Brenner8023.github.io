    # git学习笔记

    参考来源:  Git recipes in Chinese https://github.com/geeeeeeeeek/git-recipes 廖雪峰git教程 https://www.liaoxuefeng.com/wiki/0013739516305929606dd18361248578c67b8067c8c017b000

    # 简介

    git官网: https://git-scm.com/

    • git是目前世界上被广泛使用的现代软件版本管理系统.
    • git本身也是一个成熟并处于活跃开发状态的开源项目, 它最初是由Linux内核的创造者Linus Torvalds在2005年创造.
    • git并不是将代码的所有修改历史保存在中心服务器中. 在git中取而代之的是, 所有参与项目的开发者都拥有各自的代码完全拷贝, 并在自己的拷贝上进行软件开发.

    # 基础操作

    本地创建ssh key, 登录github, 添加rsa公钥内容
    ssh-keygen -t rsa -C "liaoxuefeng@163.com"

    克隆仓库

    # 克隆本地仓库
git clone /path/to/repository

# 克隆远端仓库(通过ssh)
git clone username@host:path/to/repository

# 克隆远端仓库(通过https)
git clone https://github.com/geeeeeeeeek/git-recipes.git

    创建新仓库git init

    • 将已存在但还没有版本控制的项目转换成一个git仓库,
    • 或者创建一个空的新仓库. 这会在你项目的根目录下创建一个新的.git目录, 其中包含了你项目的所有元数据.
    • 除了.git目录外, 已经存在的项目不会被改变
    # 将仓库连接到某个远程仓库, 取别名origin
git remote add origin

# 列出你和其他仓库之前的远程连接
git remote -v

# 删除别名为origin的远程仓库连接
git remote rm origin

# 重命名
git remote rename old-name new-name

    添加与提交

    # 添加到缓存区
git add test.js
git add *

# 查看工作目录和缓存区的状态
git status

# 把改动提交到HEAD
git commit -m "代码提交信息"

# 把改动推送给远端仓库
git push origin master

# 将远端仓库origin的内容拉下来
git pull origin master

    配置

    # 定义当前仓库所有提交使用的作者姓名
git config user.name "brenner"

# 定义当前用户所有提交使用的作者姓名
git config --global user.name "brenner"

# 定义当前用户所有提交使用的作者邮箱
git config --global user.email "brenner@qq.com"

    # 和远端仓库之间的连接

    当你用git clone克隆远程仓库时, 它自动创建了一个名为origin的远程连接

    # 将仓库连接到某个远程仓库, 取别名origin
git remote add origin

# 列出你和其他仓库之前的远程连接
git remote -v

# 删除别名为origin的远程仓库连接
git remote rm origin

# 重命名

    # 分支

    创建分支
    创建一个属于你自己的分支, 别人看不到, 还继续在原来的分支上正常工作, 而你在自己的分支上干活. 直到开发完毕, 再一次性合并到原来的分支上

    # 创建分支
git branch dev

# 切换分支
git checkout dev

# 创建dev分支, 然后切换到dev分支
git checkout -b dev

# 查看当前分支
git branch

# 合并指定分支到当前分支
git merge dev

# 删除分支
git branch -d dev

    # git log

    # 显示完整的项目历史
git log

# 限制显示提交的数量
git log -n 1

# 将每个提交压缩到一行
git log --oneline

# 显示代表每个提交的一堆信息
git log -p

# 搜索特定作者的提交, 可以使用正则
git log -author="brenner"

# 搜索提交信息匹配待定<pattern>的提交, 可以使用正则
git log -grep="<pattern>"

# 只显示包含特定文件的提交, 查找特定文件的历史
git log REANME.md

    # 回滚错误的修改

    git reflog
    用来记录你的每一次命令

    git revert

    • 用来撤销一个已经提交的快照.
    • 但是, 它是通过搞清楚这个提交引入的更改,
    • 然后在最后加上一个撤销了更改的新提交, 而不是从项目历史中移除这个提交. 这避免了git丢失项目历史
    git commit -m "Make some changes that will be undone"
git revert HEAD

    git reset
    是一个撤销更改的危险的方式, 这个撤销是永远的

    # .gitignore

    忽略特殊文件

    https://github.com/github/gitignore

    忽略文件的原则:

    1. 忽略操作系统自动生成的文件, 比如缩略图等
    2. 忽略编译生成的中间文件, 可执行文件
    3. 忽略你自己的带有敏感信息的配置文件 前端gitignore
