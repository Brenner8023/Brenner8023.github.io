    # 构建工具

    Web前端技术的应用范围越来越广泛. 从复杂庞大的管理后台到对性能要求苛刻的移动网页, 再到类似ReactNative的原生应用开发方案. 通过直接编写js, css, html开发web应用已经无法应对当前web应用的发展.

    前端技术快速发展, 各种可以提高开发效率的新思想和框架被发明. 但是这些东西都有一个共同点: 源代码无法直接运行, 必须通过转换后才可以正常运行.

    构建工具的作用就是做转换这件事情

    • 代码转换: TypeScript, es6编译成es5, scss编译成css
    • 文件优化: 代码压缩, 图片合并压缩
    • 模块合并: 把模块分类合并成一个文件
    • 自动刷新: 监听本地源代码的变化, 自动重新构建, 刷新浏览器
    • 代码校验: 校验代码是否符合规范

    构建其实是工程化, 自动化思想在前端开发中的体现, 构建工具解放了前端开发的生产力.

    在Webpack里一切文件皆模块, 通过loader转换文件, 通过plugin注入钩子, 最后输出由多个模块组合成的文件.
    webpack是基于入口, 会自动地递归解析入口所需要加载的所有资源文件, 然后用不同的Loader来处理不同的文件, 用Plugin来扩展webpack的功能

    安装

    npm install -D webpack webpack-cli

    Webpack在执行构建时默认会从项目根目录下的webpack.config.js文件读取配置.

    const HtmlWebpackPlugin = require("html-webpack-plugin");
const path = require("path");
const CleanWebpackPlugin = require("clean-webpack-plugin"); // 打包前清除旧文件
const ExtractTextWebpackPlugin = require("extract-text-webpack-plugin"); // 抽离css

module.exports = {
    entry: "./src/main.js",
    output: {
        filename: "[name].[hash:6].js",
        path: path.resolve(__dirname, 'dist')
    },
    mode: "development",
    module: {
        rules: [
            {
              test: /\.css$/,
              use: 
              	ExtractTextWebpackPlugin.extract({
                	fallback: "style-loader", // 使用style-loader抽离css
                	use: ["css-loader"] // 使用css-loader编译
            		})
            },
            {
              test: /\.(jpg|png|svg)$/, 
              use: [{
                loader: "file-loader",
                options: {
                    limit: 8192, // 小于8k的图片自动转成base64格式
                    outputPath: "images"
                }
            	}]
            },
            {test: /\.(js|jsx)$/, exclude: /node_modules/, loader: "babel-loader"}
        ]   
    },
    plugins: [
        new HtmlWebpackPlugin(
            {template: "./src/index.html"}
        ),
        new CleanWebpackPlugin(),
        new ExtractTextWebpackPlugin("style.css")
    ],
    devServer: {
        contentBase: path.resolve(__dirname, 'dist'),
        port: 9000
    }
}

    # 构建功能

    # 使用Loader

    Webpack不原生支持解析css文件. 要支持非js类型的文件, 需要使用Webpack的Loader机制.
    配置文件里的module.rules数组配置了一组规则, 告诉Webpack在遇到哪些文件时使用哪些Loader去加载和转换.
    use属性的值需要是一个由Loader名称组成的数组, Loader的执行顺序是由后到前的
    每一个Loader都可以通过URL querystring的方式传入参数, 例如css-loader?minimize中的minimize告诉css-loader要开启css压缩

    const path = require('path');

module.exports = {
  entry: './main.js',
  output: {
    // 把所有依赖的模块合并输出到一个 bundle.js 文件
    filename: 'bundle.js',
    // 输出文件都放到 dist 目录下
    path: path.resolve(__dirname, './dist'),
  },
  module: {
    rules: [
      {
        // 用正则去匹配要用该 loader 转换的 CSS 文件
        test: /\.css$/,
        use: ['style-loader', 'css-loader?minimize'],
      }
    ]
  }
};

    # 使用Plugin

    ExtractTextPlugin插件的作用是提取出JavaScript代码里的css到一个单独的文件.

    const path = require('path');
const ExtractTextPlugin = require('extract-text-webpack-plugin');

module.exports = {
  entry: './main.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, './dist'),
  },
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ExtractTextPlugin.extract({
          // 转换 .css 文件需要使用的 Loader
          use: ['css-loader'],
        }),
      }
    ]
  },
  plugins: [
    new ExtractTextPlugin({
      // 从 .js 文件中提取出来的 .css 文件的名称
      filename: `[name]_[contenthash:8].css`,
    }),
  ]
};

    # 配置

    entry配置模块的入口

    output
    配置如何输出最终想要的代码
    output.filename配置输出文件的名称, 为string类型.
    output.path配置输出文件存放在本地的目录, 必须是string类型的绝对路径
    output.publicPath配置发布到线上资源的url前缀, 为string类型.
